> 6,384 次提交。约 587,000 行代码。9 个月。一个人带着 AI 工具写出来的 agent 框架。
>
> 我们用另一个 AI，花了两天、烧掉 1.36 亿 token，把它从头到尾拆开看了一遍。

这是对 [NousResearch/hermes-agent](https://github.com/NousResearch/hermes-agent) v0.11.0 的完整源码分析。不是 API 手册，不是使用教程，而是**架构拆解**——像拆开一台精密机器，看每个齿轮为什么在那里、和谁咬合、缺了会怎样。

12 篇叙事风格的技术文档。每篇都回答一个问题：**这个子系统面临什么问题，做了什么选择，为什么不选别的方案，出问题了会怎样。**

## 分析对象

Hermes Agent 是 Nous Research 开源的自改进 AI 智能体框架。它同时是四个东西：一个可用的 AI 助手、一个支��� 19 个消息平台的网关、一个 66 个工具的执行引擎、以及一个 RL 训练数据工厂。

有意思的是——它自己大概率也是用 AI 写出来的（[证据在第 00 篇](docs/00-项目概述.md#这些代码是人写的吗)）。

## 这份分析是怎么做出来的

578,000 行代码不可能靠一个人（或一个 AI）一次性看完不出错。我们用三个 AI Agent 分工协作——一个负责读代码写草稿，两个负责挑毛病：

```
分析 Agent (sonnet)     → 读源码，产出草稿
  ↓
[事实审核 Agent ‖ 文学性审核 Agent]  → 并行审核，互不干扰
  ↓
主线 (opus)             → 二次验证 → 修正 → 增量复核 → 定稿
```

整个过程消耗约 **1.36 亿 token**，历时 **两天**。每个概念用 9 问模板分析（是什么 / 从哪来 / 在哪里 / 依赖谁 / 怎么工作 / 解决什么 / 为什么不用别的 / 出错了会怎样 / 哪里可以调），织入 Martin Fowler 风格的叙事中。

方法论不是一开始就有的——它在工作过程中从用户的反馈里逐步长出来。完整的演进记录在[工作日志](docs/99-工作日志.md)里。

## 五个最有意思的发现

1. **这个项目大概率是 AI 写的**。主要贡献者日均 53 次提交，凌晨比白天活跃，131 次 Claude Co-Author 标记。我们推测 60-80% 的代码行数由 AI 生成——而这个项目本身就是一个 AI agent。用 AI 开发 AI，dogfooding 到了极致。

2. **两个"上帝文件"扛起了整个系统**。`run_agent.py`（13,293 行）和 `cli.py`（11,395 行）——加起来快 25,000 行。`agent/` 子目录是逐步从 `run_agent.py` 抽离出去的，但"胶水"本身也有 13,000 行。

3. **安全不是事后补丁，是 8 层纵深防御**。从硬核封锁（`rm -rf /` 无条件拒绝）到 LLM 自动判断的 smart 模式，从 SSRF 防护到 Tirith 内容扫描——即使开了 `--yolo`，最危险的命令仍然被拦截。

4. **Prompt Caching 的守护比 Caching 本身更精巧**。标记 4 个 breakpoint 只需要 30 行代码，但为了让缓存真正命中——系统提示只构建一次、JSON sort_keys 标准化、动态信息走用户消息而非系统提示——这些"守护策略"分散在几千行代码里。

5. **它真的能自己训练下一代自己**。`batch_runner.py` 批量生成轨迹 → `trajectory_compressor.py` 压缩到训练窗口 → `environments/` 提供 Atropos RL 环境 → 11 种工具调用格式解析器确保跨模型兼容。这不是概念验证，是可以跑的训练管线。
