## 核心特性

| 特性 | 描述 | 来源 |
|------|------|------|
| 闭环学习 | 自动创建技能、技能自改进、FTS5 会话搜索 + LLM 摘要跨会话召回、Honcho 用户建模 | `README.md:21` |

| 调度自动化 | 内置 cron 调度器，支持自然语言任务、定期报告 | `README.md:22` |
| 并行子代理 | 支持 spawn 隔离子代理并行处理工作流 | `README.md:23` |
| 多终端后端 | 本地、Docker、SSH、Daytona、Singularity、Modal 六种终端后端 | `README.md:24` |
| 研究就绪 | 批量轨迹生成、Atropos RL 环境、轨迹压缩，用于训练工具调用模型 | `README.md:25` |
| 模型无关 | 支持 OpenRouter、Anthropic、OpenAI、Nous Portal、Gemini、NVIDIA NIM 等 200+ 模型 | `README.md:16` |

## 项目结构

### 核心模块

| 路径 | 用途 | 确认来源 |
|------|------|----------|
| `hermes_cli/` | 统一 CLI 入口（`hermes` 命令），含所有子命令 | `hermes_cli/__init__.py:2-12` |
| `agent/` | Agent 核心模块（提示词构建、上下文压缩、记忆管理、多 provider 适配） | `agent/__init__.py:1-6` |
| `tools/` | 全部工具实现（66 个 .py 文件，覆盖终端/文件/Web/浏览器/语音等） | `tools/__init__.py:1-15` |
| `gateway/` | 多平台消息网关（会话管理、配置、交付路由） | `gateway/__init__.py:1-10` |
| `cron/` | Cron 调度系统（定时任务创建/执行/管理） | `cron/__init__.py:1-16` |
| `plugins/` | 可选插件（context_engine/memory/image_gen/spotify 等） | `plugins/` 目录 |
| `skills/` | 预置技能库（productivity/creative 等） | `skills/` 目录 |

## 配置方式

配置文件：`cli-config.yaml`（从 `cli-config.yaml.example` 复制）

主要配置段：

| 配置段 | 行号范围 | 功能 |
|--------|----------|------|
| `model:` | 8-66 | 模型选择（provider、api_key、base_url、context_length） |
| `terminal:` | 148-237 | 终端后端（local/ssh/docker/singularity/modal/daytona） |
| `compression:` | 306-327 | 上下文压缩（threshold/target_ratio） |
| `memory:` | 409-427 | 持久记忆（memory_enabled/char limits） |
| `skills:` | 481-493 | 技能系统 |
| `agent:` | 499-551 | Agent 行为（max_turns/reasoning_effort） |
| `platform_toolsets:` | 600-609 | 每平台工具集 |
| `mcp_servers:` | 707-735 | MCP 服务端接入 |
| `hooks:` | 989-1001 | Shell 脚本 hook |

配置优先级（`cli.py:1985`）：**CLI flag > 环境变量 > yaml 配置文件 > 内置默认值**。

HERMES_HOME 路径：默认 `~/.hermes`，可通过 `HERMES_HOME` 环境变量覆盖（`hermes_constants.py:17-18`）；Docker 中为 `/opt/data`（`Dockerfile:64`）。
