# 种子测试结果

> 时间：2026-05-24
> 目的：验证事实审核 Agent 和文学性审核 Agent 的有效性

## 测试方法

写一篇故意埋入 4 个事实错误 + 3 个文学性问题的短文档，并行启动两个 sonnet 审核 Agent，检验是否能抓到。

## 埋入的错误

### 事实错误（4 个）
1. `CredentialPool` 行号 250（实际 389）
2. 冷却机制描述为"指数退避，30 秒起步，上限 15 分钟"（实际是固定时长：401→5 分钟，429→1 小时）
3. `_SUMMARY_RATIO = 0.15`（实际 0.20）
4. `DELEGATE_BLOCKED_TOOLS` 包含 6 个工具含 `web_search`（实际 5 个，无 web_search）

### 文学性问题（3 个）
1. 最后一段塞入 11 个技术概念，信息密度过高
2. "凭证池" vs "认证池" 术语不一致
3. 第 2 节→第 3 节缺过渡，硬跳转

## 审核结果

### 事实审核 Agent（sonnet）
- **4/4 事实错误全部抓到**（2❌ + 2⚠️）
- 额外发现：SUMMARY_PREFIX 简写遗漏关键部分（⚠️）、状态数不对（⚠️）
- 9 问覆盖检查：3 个概念均有大量缺项
- 图表缺失：标记 3 处应加图
- 反橡皮图章：通过（列出 3 个"最可能出错但实际正确的断言"）

### 文学性审核 Agent（sonnet）
- **3/3 文学性问题全部抓到**
- 额外发现：概念引入节奏问题、比喻顺序建议、文风不一致
- 共 7 个改进建议（3 必须改 + 4 建议改）
- 反橡皮图章：通过（远超 3 个最低建议数）

## 结论

两个审核 Agent 均有效，可以投入正式使用。
