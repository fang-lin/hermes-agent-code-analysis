# reviews/ 目录说明

**权威审核记录在 [`../98-审核报告汇总.md`](../98-审核报告汇总.md)**，本目录不是。

## 这个目录里有什么

只保留两类一次性方法论产物：

- `seed-test-results.md` — v2 启动时的种子测试结果，验证两个审核 Agent 能抓出故意埋入的事实/文学性错误。
- `completeness-phase1.md` — 阶段 1 完整性审核的原始报告。（阶段 2/3 的记录在工作日志 99 与 98 报告中，未单独落盘。）

## 为什么没有逐章的原始审核报告

WORKFLOW.md 早期设计要求把每章的深度/文学性/事实审核原始报告落盘到本目录（`XX-depth/literary/factual-review.md`）。实践中这个动作只在 00–02 章执行过（2026-05-24 那几个 session），从 03 章起就改为**把审核结论直接汇总进 `98-审核报告汇总.md`**——统计 + 修正项表格 + 二次验证结论。

2026-05-29 决定：认定 98 报告为权威且完整的审核记录，删除 00–02 那批半套原始报告（它们会误导人以为"后面的章节没审"），并将 WORKFLOW.md 步骤 3/5/7 对齐为"汇总进 98 报告"。原始 Agent 报告是中间产物，不再要求落盘。

需要查某章审了什么、改了什么 → 看 `98-审核报告汇总.md` 对应章节小节，过程叙述看 `99-工作日志.md`。
