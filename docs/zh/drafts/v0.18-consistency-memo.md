### 统一数字口径（均已亲核，可复现）

- **Platform 枚举 ≠ 平台数**：`gateway/config.py` 的 Platform 枚举 24 项，含 `local`/`relay`/`wecom_callback` 等非适配器项，且不含 irc/line/teams 等插件平台名——**不要用枚举数当平台总数**

### 加载机制（写入 01/05/07 章的统一表述）

1. 插件发现时**不 import 平台模块**，只注册轻量延迟加载器：`hermes_cli/plugins.py:1445` → `_register_deferred_platform`（:1705）→ `platform_registry.register_deferred`（:1730）
2. 动机（`gateway/platform_registry.py:174-182` 注释原文）：平台适配器在模块级 import 重 SDK（lark_oapi、slack_bolt、discord.py…），急加载 ~20 个插件平台曾给**每次** `hermes` 调用加数秒启动延迟，包括根本不碰 gateway 的 `hermes chat`
3. 首次真实查询（gateway 启动 / cron 投递 / `hermes setup` / send_message）才触发 `_resolve()` import 真模块，模块内调 `ctx.register_platform()`（`hermes_cli/plugins.py:929`）注册 `PlatformEntry(source="plugin")`
4. gateway 建适配器：`_create_adapter`（`gateway/run.py:8514`）**先查 platform_registry**（插件平台），注册表命中但实例化失败**不回落**直接返回 None；没注册才落入内建 if/elif
5. `PlatformEntry` 携带的不只是工厂：required_env / setup_fn / allowed_users_env / max_message_length / pii_safe / platform_hint（注入系统提示）/ env_enablement_fn 等（`gateway/platform_registry.py:39-120`）——`hermes setup`、授权检查、消息分块都从注册表读元数据

### 各章落点

## 包②：moa 重构（波及 02 / 03 / 06 / 12）

### 定论

**"moa 工具 → MoA 虚拟模型/循环模式"假设成立**，且是单个 commit 完成的原子重构：
`c6575df92` "feat(moa): expose MoA presets as selectable virtual models (#46081)" 同时删除 `tools/mixture_of_agents_tool.py`、新增 `agent/moa_loop.py`（1,073 行）/ `agent/moa_trace.py` / `hermes_cli/moa_cmd.py` / `hermes_cli/moa_config.py`。新代码库中 `mixture_of_agents` 零引用（tools/、agent/ 均已核）。

### 新机制要点（均已亲核）

- **设计立场**（`agent/moa_loop.py:1-7` 模块注释原文）：`/moa` 刻意**不做成模型工具**——它把一个用户轮标记为 MoA-enabled；正常 agent 循环仍拥有工具调用与轮次终止权，moa_loop 只在每次模型迭代前收集参考模型意见
- **触发方式三种**：
  1. `/moa <prompt>` 一次性糖（`cli.py:8791`）：切到虚拟 provider `"moa"`（base_url `moa://local`），跑一轮后自动还原原模型（`_pending_moa_disable_after_turn`；gateway 侧同样有 one-shot restore）
  2. `/model moa:<preset>` 显式切换，或裸 `/model <preset>` 隐式匹配（`hermes_cli/model_switch.py` PATH B；隐式匹配尊重 preset 的 `enabled: false` 退出开关，`moa_config.py:213-233`）
  3. 文本-only 前端：`encode_moa_turn` 把配置编进消息（marker `__HERMES_MOA_TURN_V1__`，`moa_config.py:10`），`agent/conversation_loop.py:555-562` 用 `decode_moa_turn` 解出
- **运行时**：`conversation_loop.py:853-874` 调 `aggregate_moa_context`（moa_loop.py）→ 并发扇出参考模型咨询调用（上限 `_MAX_REFERENCE_WORKERS = 8`，moa_loop.py:27），结果拼进用户消息；聚合者=主模型
- **计费独立**：`_RefAccounting`（moa_loop.py:30-46）——每个 advisor 按**自己模型的费率**计价，不折进聚合者用量（注释明说折算会"misprice every advisor"）
- **默认 preset**（`moa_config.py:13-21`）：参考模型 openai-codex/gpt-5.5 + openrouter/deepseek-v4-pro，聚合者 openrouter/anthropic/claude-opus-4.8
- **配置管理**：`hermes moa` 子命令（moa_cmd.py `cmd_moa`）；preset 存 config `moa.presets`

### 各章落点

- **02**：新增 MoA 循环小节（moa_loop/moa_trace/conversation_loop 注入点）——机制主讲章
- **03**：mixture_of_agents 从工具清单移除（工具总数 72→69 的组成部分）；可留一句"重构为 MoA 循环模式，见 02 章"

- **12**：8 个分布的 moa 项删除、reasoning 分布重构——原因一句话带过（工具不存在了），指向 02 章
