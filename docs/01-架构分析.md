## 1. 请求处理流程

### 1.1 CLI 主循环

```
cli.py:10819  def process_loop():
cli.py:10820      while not self._should_exit:
cli.py:10824          user_input = self._pending_input.get(timeout=0.1)
```

流程：
1. `HermesCLI.run()` (`cli.py:9213`) 启动，展示 banner，初始化 `prompt_toolkit` Application
2. `process_loop()` (`cli.py:10819`) 在主线程运行，轮询 `_pending_input` 队列（100ms timeout）
3. 用户输入经 `prompt_toolkit` TextArea 放入队列
4. 检测到输入后：文件拖拽检测 (`cli.py:10861`)、斜杠命令分发 (`cli.py:10876`)、或调用 `self.chat()` (`cli.py:10903`)

### 1.2 chat() → Agent 线程

`chat()` 定义于 `cli.py:8404`，在后台 daemon 线程 (`cli.py:8664`) 中调用 Agent：

```python
# cli.py:8636
result = self.agent.run_conversation(
    user_message=agent_message,
    conversation_history=self.conversation_history[:-1],
    stream_callback=stream_callback,
    task_id=self.session_id,
)
```

### 1.3 AIAgent.run_conversation() 主循环

**入口**：`run_agent.py:9627`

**完整调用链**：

```
用户输入
  → cli.py:10903  self.chat(user_input)
    → cli.py:8636  agent.run_conversation(...)
      → run_agent.py:9798  messages.append({"role":"user","content":user_message})
      → run_agent.py:9814  构建/缓存系统提示词 (_cached_system_prompt)
      → run_agent.py:9989  记忆预取 (_memory_manager.prefetch_all)
      → run_agent.py:9993  while 循环:
          api_call_count < max_iterations
          AND iteration_budget.remaining > 0
          OR _budget_grace_call
          → run_agent.py:10348  构建 API kwargs
          → run_agent.py:10412  流式 API 调用
          → 解析响应:
            若 finish_reason == "tool_calls":
              → run_agent.py:8594  _execute_tool_calls()
                → run_agent.py:8636  _invoke_tool(name, args)
                  → model_tools.py:500  handle_function_call()
                    → tools/registry → handler(args)
              → 工具结果 append 到 messages → 继续循环
            若 finish_reason == "stop":
              → 退出循环，返回 final_response
```

## 2. 多 Provider 适配架构

### 2.1 api_mode 选择

**位置**：`run_agent.py:982-1013`（`AIAgent.__init__` 内）

| 条件 | api_mode |
|------|----------|
| 显式传入 `api_mode` | 直接使用 |
| `provider == "openai-codex"` 或 `"xai"` | `codex_responses` |
| `provider == "anthropic"` 或 `base_url` 含 `api.anthropic.com` | `anthropic_messages` |
| `provider == "bedrock"` 或 `base_url` 含 `bedrock-runtime.` | `bedrock_converse` |
| 其他（默认） | `chat_completions` |

### 2.2 Transport 抽象层

| `convert_tools()` | base.py:35 | OpenAI tool schema → provider 格式 |
| `build_kwargs()` | base.py:43 | 组装完整 API 调用参数 |
| `normalize_response()` | base.py:60 | provider 响应 → `NormalizedResponse` |

**注册机制**（`transports/__init__.py:14`）：
```python
_REGISTRY: dict = {}
```

### 2.3 Transport 实现

| api_mode | Transport 类 | 底层 Adapter |

## 4. 工具注册与调度

### 4.1 ToolRegistry

**位置**：`tools/registry.py:100`

工具文件在模块级别调用 `registry.register()` 自注册：
```python
registry.register(
    name="read_file",
    toolset="file",
    schema=READ_FILE_SCHEMA,
    handler=_handle_read_file,
    ...
)
```

**发现机制**（`tools/registry.py:56`）：`discover_builtin_tools()` 用 AST 扫描所有 `tools/*.py`，找到有 `registry.register()` 调用的模块后 import，触发自注册。

**触发点**：`model_tools.py:139` 在模块级调用 `discover_builtin_tools()`。

### 4.2 工具调度

**入口**：`model_tools.py:500` — `handle_function_call()`

```
_invoke_tool(function_name, function_args)       # run_agent.py:8636
  → handle_function_call(name, args, task_id)    # model_tools.py:500
    → registry 查表 → entry.handler(args, **kw)  # tools/*.py
```

### 4.3 Toolset 配置

核心工具集 `_HERMES_CORE_TOOLS`（`toolsets.py:31`）是所有平台共享的默认工具列表。

`get_tool_definitions()`（`model_tools.py:209`）按 enabled/disabled toolsets 过滤，返回 OpenAI 格式 schema 列表。

## 6. 子代理（Delegation）架构

**位置**：`tools/delegate_tool.py`

### 6.1 入口

`delegate_task()`（`delegate_tool.py:1813`）支持两种模式：
- **Single**：`goal` + optional `context`/`toolsets`/`role`
- **Batch**：`tasks` array，最多 `max_concurrent_children` 个并发

### 6.2 安全约束

**禁用工具**（`delegate_tool.py:41-49`）：
```python
DELEGATE_BLOCKED_TOOLS = frozenset([
    "delegate_task",   # 禁止递归
    "clarify",         # 禁止用户交互
    "memory",          # 禁止写 MEMORY.md
    "send_message",    # 禁止跨平台副作用
    "execute_code",    # 子代理应逐步推理
])
```

**深度控制**（`delegate_tool.py:129`）：
```python
MAX_DEPTH = 1          # parent(0) → child(1)，孙子被拒绝
_MAX_SPAWN_DEPTH_CAP = 3  # 可配置上限
```

**工具集继承**：子代理最多拥有父代理的工具子集（取交集），不能获得父没有的工具（`delegate_tool.py:891-930`）。

**审批处理**：子代理线程无 CLI stdin，通过 `_subagent_auto_deny`（默认）或 `_subagent_auto_approve` 处理危险命令（`delegate_tool.py:69-107`）。

### 6.3 可观测性

```python
_active_subagents: Dict[str, Dict]    # 运行中子代理注册表
list_active_subagents()               # TUI 可查看
interrupt_subagent(subagent_id)       # 中断特定子代理
set_spawn_paused(paused)              # 全局暂停新 spawn
```

## 8. 关键设计模式

| 模式 | 位置 | 说明 |
|------|------|------|
| Transport ABC | `agent/transports/base.py:16` | 4 方法接口隔离 provider 差异 |
| 自注册 Registry | `tools/registry.py:100` | 工具文件 import 时注册，model_tools 统一查询 |
| Lazy Import | `run_agent.py:7517`, `delegate_tool.py:866` | 避免循环依赖，按需加载 |
| 系统提示缓存 | `run_agent.py:9814` | 跨 turn 不变，最大化 Anthropic prefix cache |
| 队列化中断 | `cli.py:10824` | 主循环 vs agent 线程通过 Queue 解耦 |
| 子代理隔离 | `delegate_tool.py:41-49` | blocked tools + 工具集交集继承 |
| Gateway Agent 缓存 | `gateway/run.py:41` (128 上限), `run.py:709` (实例化) | LRU + TTL 复用 Agent |
| Memory 双注入 | `run_agent.py:4629` + `run_agent.py:10131` | 静态快照入 system prompt；动态检索入 user message |
